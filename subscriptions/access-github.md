# <a name="introducing-visual-studio-subscriptions-with-github-enterprise"></a>GitHub Enterprise ile Visual Studio abonelikleriyle tanıtımı  

Microsoft ile Kurumsal Anlaşmalar (EA) olan müşteriler, Visual Studio standart aboneliklerini ve GitHub Enterprise'ı bir araya getiren yeni bir abonelik teklifi satın alabilir. Visual Studio abonelerinin GitHub Enterprise'ı satın alması kolay ve ekonomik bir yoldur. 

- IDE ve iş akışlarınıza açık kaynak kodu, topluluk ve en iyi uygulamaları güvenli bir şekilde getirerek Visual Studio ve GitHub ile ölçekte yenilik yapın.
- Harika uygulamalar oluşturmak, dağıtmak ve yönetmek için en kapsamlı kaynak kümesine sahip geliştiriciler için sınıfı geliştirme araçlarını ve hizmetlerini en iyi şekilde kullanın. 
- İş birliği yapın, GitHub Enterprise'daki çekme isteklerini yönetin ve kaynak kodunu yazıldığı yerde - editörde gözden geçirin. 

## <a name="whats-available"></a>Neler var? 

Kuruluşunuz yeni bir EA giriyorsa veya varolan bir EA'yı yeniliyorsa, aşağıdaki seçenekler kullanılabilir:

- GitHub Enterprise ile Visual Studio Enterprise
- GitHub Enterprise ile Visual Studio Professional

Kuruluşunuzun henüz yenilenmeyen bir EA'sı varsa, GitHub Enterprise'ı mevcut Visual Studio Enterprise veya Visual Studio Professional aboneliklerinize eklemek için "Adım Adım" SKU'yu satın alabilirsiniz. Ardından, anlaşmanız yenilendiğinde, teklifi satın alabilirsiniz.

Visual Studio Enterprise, Visual Studio Professional ve GitHub Enterprise için mevcut SKU'lar da ayrı olarak kullanıma sunulmaya devam edecektir. 

Avantajlar ve seçenekler hakkında daha https://visualstudio.microsoft.com/subscriptions/visual-studio-githubfazla bilgi edinmek için lütfen ziyaret edin. 

## <a name="getting-your-subscriptions"></a>Aboneliklerinizi alma

Visual Studio + GitHub Enterprise abonelikleri iki bölümden oluşur:
1. Visual Studio aboneliği (Kurumsal veya Profesyonel)
2. GitHub Kurumsal aboneliği

Kuruluşunuz abonelik satın aldığında, bunlar yöneticiler tarafından atanır. Visual Studio abonelikleri ve GitHub Enterprise abonelikleri ayrı olarak atanır.  

Size bir Visual Studio + GitHub aboneliği atanmışsa, Visual Studio abonelik yöneticinizin size bir abonelik atadığını bildiren bir e-posta alırsınız.  Bu noktada, onu https://my.visualstudio.com/subscriptionsgörebilirsiniz.  

GitHub Kurumsal abonelikleri ayrı olarak atandığından, şirketinizin aboneliklerini nasıl düzenlediğine bağlı olarak farklı bir yöneticiden bildirim alabilirsiniz.  Visual Studio aboneliğinin atanması ile GitHub Enterprise aboneliği arasında da bir gecikme olabilir, bu nedenle her ikisi de hemen kullanılamıyorsa endişelenmeyin.  GitHub abonelik atama işlemi daha karmaşıktır ve tamamlanması birkaç iş günü sürebilir.

## <a name="frequently-asked-questions"></a>Sık sorulan sorular

### <a name="q--where-is-my-github-subscription"></a>S: GitHub aboneliğim nerede?
C: GitHub aboneliğini ayarlama işlemi biraz zaman alabilir.  Bekliyorsanız ve hiçbir şey duymadıysanız, GitHub yöneticinize ulaşmak isteyebilirsiniz.  Yöneticiniz hala üzerinde çalışıyor veya size bir GitHub aboneliği atamamayı seçmiş olabilir. 

### <a name="q--how-do-i-reach-my-github-administrator"></a>S: GitHub yöneticime nasıl ulaşabilirim?
C: GitHub Enterprise Server kullanıyorsanız, yöneticinizle bağlanmak için istemcinin alt kısmında görünen **Destek** bağlantısını tıklatabilirsiniz.

### <a name="q-do-i-get-the-full-visual-studio-subscription-professional-or-enterprise-or-do-i-just-get-the-ide-with-this-offering"></a>S: Visual Studio aboneliğinin tamamını (Professional veya Enterprise) alabilir miyim yoksa bu teklifle IDE'yi mi alabilirim?
C: GitHub Enterprise aboneliğinin yanı sıra tam bir Visual Studio aboneliği alırsınız.  Bu aboneliklerde yer alan yazılım indirme, bulut hizmetleri, teknik destek ve iş ortağı avantajları gibi tüm avantajları elde edersiniz, tıpkı aboneliğin ayrı olarak satın alınmış sayılsa bile.

# <a name="signing-in-to-your-visual-studio-subscription"></a>Visual Studio aboneliğinizde oturum açma

Adımlar Visual Studio aboneliğinizde oturum açmak için kullandığınız hesap türüne bağlıdır.  Örneğin, bir Microsoft hesabı (MSA) veya sizin İşveren veya okulunuz tarafından sağlanan bir e-posta adresi kullanıyor olabilir.  Ocak 2019'den itibaren artık GitHub hesabınızda oturum açmak için bazı abonelikler için kullanabilirsiniz. 

Dört farklı oturum açma yöntemleri bu makalede ele alınmıştır:.  Sağdaki bağlantılar, tüm bu bölümleri atlamak için kullanın.
1. Microsoft hesabınızla (MSA) imzalama
2. İş/Okul hesabınızla oturum açma
3. Bir iş/Okul hesabınızda oturum açmak için Microsoft Account kullanma
4. GitHub hesabınızla oturum açma

## <a name="signing-in-with-your-microsoft-account-msa"></a>Microsoft hesabınızla (MSA) imzalama
1. Lütfen [ https://my.visualstudio.com ](https://my.visualstudio.com?wt.mc_id=o~msft~docs).
2. Ayarlayın ya da Visual Studio aboneliğiniz satın size sağlanan e-posta adresi girin.

   > [!NOTE]
   > Bu adres, abonelik satın alınan ya da Visual Studio Dev Essentials için kaydolan aldığınız abone Hoş Geldiniz e-postadaki da tanımlanır. Hoş Geldiniz e-posta bulma yaşıyorsanız Lütfen gereksiz posta klasörlerinizi denetleyin.

3. Parolanızı girin.
4. Tıklayın **oturum**.
5. Bu noktada "Avantajlar" sayfası görüntülenmelidir.

### <a name="for-visual-studio-dev-essentials-users"></a>Visual Studio Dev Essentials kullanıcıları için:
Visual Studio Dev Essentials aboneliğinizi ilk kez eriştiğinizde, bir Hoş Geldiniz iletişim kutusunu görürsünüz.  Tıklayın **Onayla** programın hüküm ve koşulları kabul etmek için.

## <a name="signing-in-with-your-work-or-school-account"></a>İş veya Okul hesabınızla oturum açma

1. Lütfen [ https://my.visualstudio.com ](https://my.visualstudio.com?wt.mc_id=o~msft~docs).
2. İçin yeni Visual Studio aboneliğinin atandığı e-posta adresi girin.

   > [!NOTE]
   > Bu adres, aldığınız abone Hoş Geldiniz e-postayla da tanımlanır. Hoş Geldiniz e-posta bulma yaşıyorsanız Lütfen gereksiz posta klasörlerinizi denetleyin.

3. **Devam**'a tıklayın.
4. Kurumsal oturum açma sayfasına yönlendirilirsiniz.
5. Parolanızı girin.
6. Tıklayın **oturum açın**
7. Bu noktada "Avantajlar" sayfası görüntülenmelidir

Şimdi mavi çubuğun portalın üst kısmında görüntülenen kullanmakta olduğunuz abonelik türüne görebilirsiniz.

Şu anda seçili aboneliğinize sağ üst köşede adınızı altında de görebilirsiniz.  Yazar "gösteren:" abonelik çizgidir.  Birden fazla aboneliğiniz varsa, aşağı açılan oka tıklayın ve kullanmak istediğiniz aboneliği seçin.

## <a name="using-your-microsoft-account-to-sign-in-to-a-work-or-school-account"></a>Bir iş veya Okul hesabınızda oturum açmak için Microsoft Account kullanma

1. [https://my.visualstudio.com](https://my.visualstudio.com?wt.mc_id=o~msft~docs) kısmına gidin.
2. Yeni Visual Studio aboneliğinin atandığı e-posta adresi girin

   > [!NOTE]
   > Bu adres, abonenin Hoş Geldiniz mektubunda da tanımlanır. Hoş Geldiniz mektubunda değil alındıysa Lütfen gereksiz posta klasörlerinizi denetleyin

3. **Devam**'a tıklayın.
4. Bir karar sayfasına yönlendirilirsiniz.
    - Seçin **iş veya Okul hesabı** aboneliği bir Azure Active Directory (AAD) Kiracı ile ilişkilendirilen bir "İş veya Okul" hesabıyla ilişkiliyse.
    - Seçin **kişisel** abonelik "Kurumsal" e-posta adresiyle ilişkili, ancak bir "Kişisel" Microsoft hesabı (MSA) için de dönüştürüldü.

        > [!NOTE]
        > Bu, geçmişte Visual Studio abonelikleri (eski adıyla MSDN) kullanmış abonelerin birçok durumda olacaktır.

    - Bir yol başarısız olursa diğerini deneyin.  Abonelik yöneticileri aboneliğinizi değiştirilmiş olabilir.

5. Parolanızı girin.
6. Tıklayın **oturum**.
7. Bu noktada "Avantajlar" sayfası görüntülenmelidir.

## <a name="signing-in-with-your-github-account"></a>GitHub hesabınızla oturum açma

GitHub kimlik desteği GitHub hesabınızı Microsoft hesabınıza bağlama yeni veya var olan Microsoft hesabınız için kimlik bilgisi olarak mevcut GitHub hesabınızı kullanmanıza olanak sağlar. 

GitHub ile oturum açtığınızda Microsoft Kişisel varolan GitHub hesabınızla ilişkili herhangi bir e-posta adresi olup olmadığını eşleşen veya Kurumsal Microsoft hesabına denetler. Adres enterprise hesabınızı eşleşiyorsa, o hesabı için bunun yerine oturum açmak için istenir. Kişisel hesap adresi eşleşirse, GitHub hesabınızı bir oturum açma yöntemi, kişisel hesabınıza ekleyeceğiz.

GitHub ve Microsoft hesabı kimlik bilgilerinizle bağlandıktan sonra bu tek oturum açma her yerden kişisel bir Microsoft hesabı, gibi Azure sitelerine, Office uygulamaları ve Xbox kullanılabilir kullanabilirsiniz. Bu hesaplar, Azure Active Directory'ye Konuk için kullanılabilir oturum açmalar bir Microsoft hesabı e-posta adresi varsayılarak bir davet eşleşir.

> [!NOTE]
> Bir Microsoft hesabı ile bir GitHub kimlik bağlama Microsoft herhangi bir kod erişimi vermek değil. Azure DevOps ve Visual Studio gibi uygulamalar, kod depoları erişim gerektirdiğinde bu erişim belirli izin vermeniz istenir. 

### <a name="frequently-asked-questions"></a>Sık sorulan sorular
Aşağıdaki sık sorulan sorular sorulara oturum açmak için Visual Studio abonelikleri için GitHub hesabı kimlik bilgilerinizi kullanımına ilişkin karşılaşabilirsiniz.

#### <a name="q-i-forgot-my-github-password--how-can-i-access-my-account-now"></a>S: GitHub parolamı unuttum.  Hesabımı şimdi nasıl erişebilirim?
Y:  GitHub hesabınıza giderek kurtarabilirsiniz [parolanızı sıfırlama](https://github.com/password_reset). Veya, GitHub hesabınızın e-posta adresi girerek GitHub bağlantılı Microsoft hesabınızla kurtarabilirsiniz [hesabınızı kurtarın](https://account.live.com/password/reset).

#### <a name="q-i-deleted-my-github-account--how-can-i-access-my-microsoft-account-msa-now"></a>S: Ben GitHub Hesabımı silindi.  Microsoft hesabım (MSA) artık nasıl erişebilirim?
Y: MSA (örneğin, parola, Authenticator uygulaması veya güvenlik anahtarı) üzerinde herhangi bir diğer kimlik bilgisi yoksa, Microsoft hesabınıza bağlı e-posta adresi kullanarak kurtarabilirsiniz. Başlamak için Git [hesabınızı kurtarın](https://account.live.com/password/reset). Daha sonra oturum açma biliyoruz şekilde bir parola, hesabınıza eklemek zorunda kalırsınız. 

#### <a name="q-theres-no-sign-in-with-github-option-on-the-sign-in-page--how-can-i-use-my-github-credentials-to-sign-in"></a>S: Oturum açma sayfasında "Sign in ile GitHub" seçeneği yoktur.  Oturum açmak için GitHub kimlik bilgilerimi nasıl kullanabilirim?
Y:  GitHub yazın hesabı e-posta adresi, seçtiğiniz GitHub bağlantılı Microsoft hesabınızı oluşturduğunuzda. Biz, aramak ve oturum açma için Github'a gönderme. Veya varsa oturum açma seçenekleri bağlantı oturum açma sayfasında, kullanmak **Github ile oturum aç** bu bağlantıya tıkladığınızda görüntülenen düğme. 

#### <a name="q-i-cant-sign-in-to-some-of-my-apps-and-products-with-github--why"></a>S: Bazı uygulamalar ve ürünler GitHub ile oturum açamıyorum.  Neden?
Y:  Tüm Microsoft ürünleri GitHub.com kendi oturum açma sayfasından erişebilirsiniz — Örneğin, Xbox konsolları. E-posta adresine bağlı GitHub hesabınızı yazdığınızda, gerçekten size ait doğrulayabilmemiz bunun yerine, bir kodu bu adrese göndereceğiz. Yine de aynı hesabı için yalnızca farklı bir oturum açma yöntemi oturum açtığınız. 

#### <a name="q--ive-added-a-password-to-the-microsoft-account-i-have-linked-to-my-github-account--will-that-cause-a-problem"></a>S:  Bir parola GitHub hesabıma bağlanan Microsoft hesabına eklendi.  Bir sorun neden?
Y:  Ne demek. Bu, GitHub parolanız değiştirmez; yalnızca Microsoft hesabınızda oturum açmak için başka bir yolu da sahip olacaksınız. E-posta adresinizi kullanarak oturum her oturum açmak için Microsoft hesap parolanızla oturum veya Github'a giderek seçenek sunuyoruz. Bir parola eklemeniz gerekiyorsa, GitHub hesabınızın parolasını farklı olduğundan emin olun, kesinlikle öneririz.

#### <a name="q-i-want-to-add-the-authenticator-app-to-the-account-i-created-using-github--can-i-do-that"></a>S: GitHub kullanarak oluşturduğum hesaba kimlik doğrulayıcı uygulamasını eklemek istiyorsunuz.  Bunu yapabilir miyim?
Y:  Hiç sorun — yalnızca uygulama indirmek ve e-posta adresinizi kullanarak oturum açın. E-posta adresinizi oturum açtığınızda, seçmeniz istenir [Authenticator uygulamasını](https://go.microsoft.com/fwlink/?linkid=2090219) veya GitHub kimlik bilgileriniz olarak.

#### <a name="q-ive-enabled-two-factor-authentication-on-both-my-github-and-microsoft-accounts-msa-but-when-i-sign-in-to-my-msa-im-still-asked-to-authenticate-twice--why"></a>S: My GitHub ve Microsoft hesapları (MSA) iki öğeli kimlik doğrulama etkin, ancak benim MSA oturum açtığımda miyim yine de iki kez kimlik doğrulaması güncelleştirmem isteniyor.  Neden?
Y: İki aşamalı doğrulama var. etkin olsa bile güvenlik kısıtlamaları nedeniyle, bir tek Faktörlü doğrulama olarak GitHub ile oturum imzalama Microsoft sayar. Bu nedenle, Microsoft hesabınız için yeniden kimlik doğrulaması gerekir. 

#### <a name="q--how-can-i-tell-if-my-microsoft-account-and-github-accounts-are-linked"></a>S:  Benim Microsoft hesabınız ve GitHub hesaplarında bağlı olmadığını nasıl anlayabilirim?
Y:  (E-posta adresi, telefon numarası, Skype adı) hesabı diğer adınızı kullanarak oturum olduğunda, tüm oturum açma yöntemleri hesabınız için göstereceğiz. GitHub var. görmüyorsanız, henüz kurabilecek henüz.

#### <a name="q--how-can-i-unlink-my-microsoft-and-github-accounts"></a>S:  Benim Microsoft ve GitHub hesaplarını nasıl bağlantısını? 
Y:  Git [Güvenlik sekmesinde](https://account.microsoft.com/security) account.microsoft.com tıklatın ve **daha fazla güvenlik seçeneği** GitHub hesabınızın bağlantısını kaldırmak için. Çözme, GitHub hesabı, bir oturum açma yöntemi kaldırır ve Visual Studio'da GitHub depoları erişimi kaldırır. Burada erişimi kaldırılıyor erişim tüm ürünleri kaldırmaz için diğer Microsoft ürünleri, GitHub hesabını ayrı ayrı erişim isteğinde bulundu. Git [uygulama izinleri](https://github.com/settings/applications) uygulamalarından onayı iptal etmek için GitHub profilinizde sayfasında listelenen vardır.

#### <a name="q--i-try-to-use-my-github-account-to-sign-in-but-im-prompted-that-i-already-have-a-microsoft-identity-that-i-should-use-instead--whats-happening"></a>S:  GitHub Hesabımda oturum açarken kullandığınız deneyin ancak bunun yerine kullanması gereken Microsoft Identity zaten var olduğunu isteniyor.  Ne oluyor?
Y:  GitHub hesabınızı bir Azure Active Directory e-posta adresi varsa, bu, zaten sahip olduğunuz Azure erişebilir ve CI işlem hatları kullanarak GitHub kodunuzu çalıştırmak, Microsoft Identity anlamına gelir. Bu hesabı kullanarak Azure kaynaklarını ve derleme işlem hatlarını, kuruluş sınırları içinde kalmasını sağlar. Ancak, kişisel bir iş yapıyorsanız, böylece her zaman da erişim kazanırsınız kişisel e-posta adresi, GitHub hesabınızda koyarak öneririz. Bunu yaptıktan sonra yeniden oturum açmayı deneyin ve seçin **farklı bir e-posta adresi kullanın** iş veya Okul hesabınızda oturum açmak için istendiğinde. Bu, kişisel bir e-posta adresi kullanarak yeni bir Microsoft hesabı oluşturmanızı sağlar.

# <a name="c6260"></a>C6260
Uyarı C6260: sizeof * sizeof neredeyse her zaman hatalı, bir karakter sayısı veya bayt sayısı kullanmayı mı amaçladınız?

 Bu uyarı, iki `sizeof` işlem sonuçlarının birlikte çarpıldığını gösterir. C/C++ `sizeof` işleci, bir nesnenin kullandığı depolama bayt sayısını döndürür. Genellikle bunu başka bir `sizeof` işlemiyle çarpmak yanlış; genellikle bir nesne içindeki bayt sayısıyla veya dizideki öğelerin sayısına (örneğin, bir dizideki geniş karakter sayısına) yöneliktir.

 @No__t-0 işleciyle ilişkili, sezgisel olmayan bazı davranışlar vardır. Örneğin, C 'de, bir karakter integral türünde olduğundan, `sizeof ('\0') == 4,`. ' C++De, bir karakter sabit değerinin türü `char` ' dir, bu nedenle `sizeof ('\0') == 1`. Ancak, hem C hem C++de aşağıdaki doğru geçerlidir:

```
sizeof ("\0") == 2.
```

## <a name="example"></a>Örnek
 Aşağıdaki kod bu uyarıyı oluşturur:

```
#include <windows.h>

void f( )
{
  int i;
  i = sizeof (L"String") * sizeof (WCHAR);
  // code ...
}
```

 Bu uyarıyı düzeltmek için aşağıdaki kodu kullanın:

```
#include <windows.h>

void f( )
{
  int i;
  i= sizeof (L"String") / sizeof (WCHAR);

  /* or to get bytes */
  i = sizeof (L"String");
  // code ...
}
```

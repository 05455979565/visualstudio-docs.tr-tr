# <a name="c28303"></a>C28303
Uyarı C28303: başvuru C++ parametresi < Parameter_name > için \<annotation > üzerinde belirsiz bir `_Deref_` işleci bulundu.

 Bu uyarı, uyarı C28302 benzer ve bir parametre üzerinde fazladan `_Deref_` düzeyi kullanıldığında bildirilir.

 SAL2, başvuru parametreleriyle ilgilenirken ek bir `_Deref_` kullanılmasını gerektirmez. Bu özel açıklama, açıklanmaya yönelik olarak hangi başvuru düzeyinin açıklandığını belirsizdir. Açıklama eklenecek belirli bir nesneye başvurmak için `_At_` kullanılması gerekebilir.

## <a name="example"></a>Örnek
 Aşağıdaki kod bu uyarıyı oluşturur çünkü `__deref_out_ecount(n)` kullanımı belirsiz:

```
void ref(__deref_out_ecount(n) int **&buff, int &n)
```

 Yukarıdaki ek açıklama şu şekilde yorumlanabilir:

- tamsayıya (Of n) yönelik bir başvuru (SAL1 yorumu)

- dizi (Of n) tamtamsayılara yönelik bir işaretçiye başvuru (SAL2 yorumu)

  Aşağıdakilerden biri bu uyarıyı düzeltebilir:

```
void ref(_Out_writes_(n) int **&buff, int &n)
// or
_At_(*buff), _Out_writes(n)) void ref(int **&buff, int &n)
```

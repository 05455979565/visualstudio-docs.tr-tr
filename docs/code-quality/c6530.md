# <a name="c6530"></a>C6530
Uyarı 6530: Tanınmayan biçim dizesi stili \<adı >

> [!NOTE]
>  Bu uyarı, kaynak kodu ek açıklama dili (SAL) kullanım dışı bir sürümünü kullanan kodda oluşur. SAL'ın en son sürümünü kullanmak için kodunuzu bağlantı noktası öneririz. Daha fazla bilgi için bkz: [C/C++ kod hatalarını azaltmak için SAL ek açıklamaları kullanılarak](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).

 Bu uyarı FormatString özelliği scanf veya printf dışında bir değer kullandığını gösterir. Bu uyarı düzeltmek için kodunuzu gözden geçirin ve stil özelliği için geçerli bir değer kullanın.

## <a name="example"></a>Örnek
 Aşağıdaki kod, bir yazım hatasından nedeniyle bu uyarı oluşturur. `Style` özelliği:

```
// C
#include <CodeAnalysis\SourceAnnotations.h>
void f([SA_FormatString(Style="printfd")] char *px);

// C++
#include <CodeAnalysis\SourceAnnotations.h>
using namespace vc_attributes;
void f([FormatString(Style="printfd")] char *px);

```

 Bu uyarı düzeltmek için aşağıdaki kodda gösterildiği gibi geçerli bir stil kullanın:

```
// C
#include <CodeAnalysis\SourceAnnotations.h>
void f([SA_FormatString(Style="printf")] char *px);

// C++
#include <CodeAnalysis\SourceAnnotations.h>
using namespace vc_attributes;
void f([FormatString(Style="printf")] char *px);
```

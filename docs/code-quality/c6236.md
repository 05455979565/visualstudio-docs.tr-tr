# <a name="c6236"></a>C6236
Uyarı C6236: (\<expression > &#124; &#124; \<sıfır olmayan sabit >) her zaman sıfır olmayan bir sabit değerdir

 Bu uyarı, bir test bağlamında oluşan mantıksal veya işlemin sağ tarafında sıfır olmayan bir sabit değer olduğunu gösterir. Mantıksal olarak, bu testin gereksiz olduğunu ve güvenle kaldırılabileceği anlamına gelir. Alternatif olarak, programcı, belirli bir değer veya bayrak için test etmek üzere eşitlik (`==`), bit düzeyinde AND (`&`) veya bit düzeyinde-XOR (`^`) operatörü gibi farklı bir operatör kullanmayı amaçlı olabilir.

 Bu uyarı, sıfır olmayan sabit 1 olduğunda, derleme zamanında kod yollarının seçmeli olarak etkinleştirilmesi için kullanılması nedeniyle, ortak deyim için oluşturulmaz. Ancak, sıfır olmayan sabit değer 1 ' e (örneğin, 1 + 0) değerlendirilen bir ifadeyle biçimlendirilmişse uyarı oluşturulur.

## <a name="example"></a>Örnek
 Bu kod, uyarı C6236 nasıl görüneceğini gösterir. @No__t-0 0 olmadığından, `n || INPUT_TYPE` ifadesi her zaman sıfır değildir ve `else` yan tümcesi hiçbir zaman yürütülmez. Ancak, `INPUT_TYPE` ' dan farklı bir değere sahip bir sabittir ve bu değer karşılaştırma için değer olarak ifade edilir:

```
#define INPUT_TYPE 2
#include <stdio.h>

void f( int n )
{
   if ( n || INPUT_TYPE ) // analysis warning C6236
   {
      puts( "Always gets here" );
   }
   else
   {
      puts( "Never enters here" );
   }
}
```

 Aşağıdaki kod, `n` giriş parametresinin `INPUT_TYPE` bitinin ayarlanmış olup olmadığını test etmek için bit düzeyinde AND (`&`) işlecini kullanır:

```
#define INPUT_TYPE 2
#include <stdio.h>

void f( int n )
{
   if ( n & INPUT_TYPE ) // no warning
   {
      puts( "Bitwise-AND comparison is true" );
   }
   else
   {
      puts( "Bitwise-AND comparison is false" );
   }
}
```

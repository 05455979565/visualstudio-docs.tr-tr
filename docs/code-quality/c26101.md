# <a name="c26101"></a>C26101
Uyarı C26101: @No__t-0var > değişkeni için karşılıklı kilitleme işlemi düzgün şekilde kullanılamıyor.

 Windows API 'Leri, çeşitli karşılıklı kilitleme işlemleri sunar. Ek açıklama `_Interlocked_` bir değişkene yalnızca bir kenetlenmiş işlemle erişildiğini belirtir. Bir erişim `_Interlocked_` ek açıklaması ile tutarlı olmadığında uyarı C26101 verilir.

## <a name="example"></a>Örnek
 Aşağıdaki örnek, `_Interlocked_` sözleşmesinin ihlal edildiği için uyarı C26101 oluşturur.

```

CRITICAL_SECTION cs;
typedef struct _DATA
{
    _Interlocked_ LONG data;
} DATA;

void Safe(DATA* p)
{
    InterlockedIncrement(&p->data); // OK
}

void Unsafe(DATA* p)
{
    p->data += 1; // Warning C26101
    EnterCriticalSection(&cs);
    LeaveCriticalSection(&cs);
}
```

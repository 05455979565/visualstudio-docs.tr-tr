# <a name="c28285"></a>C28285
Uyarı C28285: ' function_name ' işlevi Için ' Annotation ' içinde sözdizimi hatası

 Kod Analizi Aracı, bu uyarıyı SAL ek açıklamasında söz dizimi hataları için raporlar.  SAL ayrıştırıcı, hatalı biçimlendirilmiş ek açıklamayı atarak kurtarılır.

## <a name="example"></a>Örnek

```cpp
// The argument '(n,2)' is malformed and will cause a C28285 warning after the _Out_writes_z_ macro is expanded.
void example_func(_Out_writes_z_((2,n)) char* buffer, int n)
{
  // ...
  buffer[n] = '\0';
}
```

Kullanılan SAL ek açıklamaların belgelerini iki kez kontrol edin ve ek açıklamayı basitleştirmeye çalışın.  Doğrudan `__declspec("SAL_begin")` gibi uygulama katmanı ek açıklamalarını kullanmamalısınız. Bu katmanı kullanıyorsanız, `_In_`/`_Out_`/`_Ret_`gibi daha yüksek katmanlara değiştirin.  Daha fazla bilgi için bkz. [Işlev parametrelerine açıklama ekleme ve dönüş değerleri](annotating-function-parameters-and-return-values.md) .

```cpp
void example_func(_Out_writes_z_(n) char* buffer, int n)
{
  // ...
  buffer[n] = '\0';
}
```

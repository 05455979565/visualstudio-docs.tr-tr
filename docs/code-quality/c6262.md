# <a name="c6262"></a>C6262

Uyarı C6262: İşlev \<sabit > baytlarının kullanır:/Analyze: stacksize @ no__t-1constant 2 > değerini aşıyor. Bazı verileri yığına taşımayı düşünün

Bu uyarı, bir işlevde önceden ayarlanmış eşiği (`constant 2`) aşan yığın kullanımının algılandığını gösterir. Bu uyarı için varsayılan yığın çerçeve boyutu, Kullanıcı modu için 16 KB, çekirdek modu için 1 KB 'tır. Yığın — Kullanıcı modunda bile — sınırlıdır ve yığın taşması özel durumuna neden olan yığının bir sayfasını yürütmek için başarısız olur. Çekirdek modunun 12 KB 'lik yığın boyutu sınırı vardır ve bu artış artamaz; Bu nedenle, çekirdek modu kodu, yığın kullanımını kararlılık olarak sınırlandırmalıdır.

Bu uyarının arkasındaki sorunu gidermek için, bazı verileri yığına ya da diğer dinamik belleğe taşıyabilirsiniz.  Kullanıcı modunda, büyük bir yığın çerçevesi bir sorun olmayabilir ve bu uyarı bastırılabilir, ancak büyük bir yığın çerçevesi yığın taşması riskini artırır. (İşlev yığını yoğun veya yinelemeli olarak kullanıyorsa büyük bir yığın çerçevesi meydana gelebilir.) Yığın taşması gerçekleşirse, ancak yalnızca sistem sınırına kadar, kullanıcı modundaki toplam yığın boyutu artırılabilir.  @No__t-1 değerini değiştirmek için **/Analyze** komut satırı seçeneğini kullanabilirsiniz, ancak bunu artırmak bir hatanın bildirilmeyecek bir riski ortaya çıkarır.

Çekirdek modu kodu için — örneğin, sürücü projelerinde — `<constant 2>` değeri 1 KB olarak ayarlanır. İyi yazılmış sürücülerde, bu değere yaklaşılarak çok az sayıda işlev olmalıdır ve sınırı aşağı doğru olarak değiştirmeniz istenebilir.  Yığın boyutunu azaltmak için Kullanıcı modu kodu için kullanılan aynı genel tekniklerin çekirdek modu koduna uyarlanabilirler.

## <a name="example"></a>Örnek

Aşağıdaki kod bu uyarıyı oluşturur çünkü `char buffer` 16.382 bayt ayırır ve `i` yerel tamsayı değişkeni, birlikte 16 KB varsayılan yığın boyutu sınırını aşacak şekilde başka 4 bayt ayırır.

```cpp
// cl.exe /c /analyze /EHsc /W4
#include <windows.h>
#define MAX_SIZE 16382

void f( )
{
    int i;
    char buffer[MAX_SIZE];

    i = 0;
    buffer[0]='\0';

    // code...
}
```

Aşağıdaki kod, bazı verileri yığına taşıyarak bu uyarıyı düzeltir.

```cpp
// cl.exe /c /analyze /EHsc /W4
#include <stdlib.h>
#include <malloc.h>
#define MAX_SIZE 16382

void f( )
{
    int i;
    char *buffer;

    i = 0;
    buffer = (char *) malloc( MAX_SIZE );
    if (buffer != NULL)
    {
        buffer[0] = '\0';
        // code...
        free(buffer);
    }
}
```

Malloc ve Free kullanımı, bellek sızıntıları ve özel durumlar bakımından birçok ekde sahiptir. Bu tür sızıntıları ve özel durum sorunlarını tamamen önlemek için, C++ standart şablon KITAPLıĞı (STL) tarafından sunulan mekanizmaları kullanın. Bunlar şunlardır [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)ve [Vector](/cpp/standard-library/vector). Daha fazla bilgi için bkz. [akıllı işaretçiler](/cpp/cpp/smart-pointers-modern-cpp) ve [ C++ standart kitaplık](/cpp/standard-library/cpp-standard-library-reference).

**Yığın boyutunu ayarlayarak bu uyarıyı düzeltmek için**

1. Menü çubuğunda **proje** > **özellikleri**' ni seçin.

     **Özellik sayfaları** iletişim kutusu görüntülenir.

2. **Yapılandırma özellikleri**' ni genişletin.

3. **C/C++** Genişlet

4. **Komut satırı** özellikleri ' ni seçin.

5. **Ek seçenekler**' de, **/Analyze: stacksize16388**ekleyin.

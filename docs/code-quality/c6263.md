# <a name="c6263"></a>C6263
Uyarı C6263: bir döngüde _alloca kullanımı; Bu, yığın ile hızlı bir şekilde taşma

 Bu uyarı, bellek ayırmak için bir döngü içinde _alloca çağırmanın yığın taşmasına neden olduğunu gösterir. _alloca yığından bellek ayırır, ancak bu bellek yalnızca çağırma işlevi çıktığında serbest bırakılır. Yığın, kullanıcı modunda bile sınırlıdır ve yığın taşması özel durumuna neden olan yığının bir sayfasını yürütmek için başarısız olur. @No__t-0 işlevi bir yığın taşması koşulunu kurtarır, bu da bir programın önemli bir özel durum hatasıyla başarısız olması yerine devam etmesine izin verir. @No__t-0 işlevi çağrılmadığından, önceki özel durumdan sonra koruma sayfası yoktur. Bir sonraki sefer bir yığın taşması olduğunda, hiç özel durum yoktur ve işlem uyarı vermeden sonlandırılır.

 Ayırma boyutu veya yineleme sayısı bilinmiyorsa, yığın taşmasına neden olabileceğinden `_alloca` ' ın bir döngü içinde çağrılmasını önleyin. Bu durumlarda, yığın belleği veya [ C++ standart kitaplık](/cpp/standard-library/cpp-standard-library-reference) sınıfları gibi diğer seçenekleri göz önünde bulundurun.

## <a name="example"></a>Örnek
 Aşağıdaki kod bu uyarıyı oluşturur:

```
#include <windows.h>
#include <malloc.h>
#include <excpt.h>
#include <stdio.h>

#define MAX_SIZE 50

void f ( int size )
{
  char* cArray;
  __try
  {
    for(int i = 0; i < MAX_SIZE; i++)
    {
      cArray = (char *)_alloca(size);

     // process cArray...
    }
  }
  __except(GetExceptionCode() == STATUS_STACK_OVERFLOW ?
              EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )
  {
     // code...
     puts("Allocation Failed");
    _resetstkoflw();
  }
}
```

 Aşağıdaki kod bu uyarıyı düzeltmek için malloc () kullanır:

```
#include <windows.h>
#define MAX_SIZE 50

void f ( int size )
{
  char* cArray;

  for(int i = 0; i < MAX_SIZE; i++)
  {
     cArray = (char *) malloc(size);
     if (cArray != NULL)
     {
       // process cArray...
       free(cArray);
     }
  }
}
```

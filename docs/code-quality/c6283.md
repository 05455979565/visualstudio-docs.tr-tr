# <a name="c6283"></a>C6283
uyarı C6283: \<variable > dizi New [] ile ayrıldı, ancak skaler delete ile silindi

 Bu uyarı yalnızca C++ kodda görünür ve çağıran işlevin dizi `new []` işleçle tutarlı bir şekilde ayrılmış olduğunu, ancak skaler `delete` işleciyle serbest bırakıldığını gösterir. Bu hata, sızıntılara, bellek bozulmalarına ve işleçlerin geçersiz kılındığı durumlarda, çökmelere neden olabilir. Bellek `new []` dizisiyle ayrılmışsa, genellikle Array `delete[]` ile serbest bırakılmalıdır.

## <a name="example"></a>Örnek
 Aşağıdaki kod bu uyarıyı oluşturur:

```
void f( )
{
  char *str = new char[50];
  // code ...
  delete str;
}
```

 Bu uyarıyı düzeltmek için aşağıdaki kodu kullanın:

```
void f( )
{
  char *str = new char[50];
  // code ...
  delete[] str;
}
```

 Uyarı C6283 yalnızca, tamsayılar veya karakterler gibi basit türlerin dizileri için geçerlidir. Dizinin öğeleri, sınıf türünde nesnelerdir, uyarı [C6278](../code-quality/c6278.md) verilir.

 Yeni ve Sil kullanımı, bellek sızıntıları ve özel durumlar bakımından birçok ekde sahiptir. Bu tür sızıntıları ve özel durum sorunlarını tamamen önlemek için, C++ standart şablon KITAPLıĞı (STL) tarafından sunulan mekanizmaları kullanın. Bunlar şunlardır [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)ve [Vector](/cpp/standard-library/vector). Daha fazla bilgi için bkz. [akıllı işaretçiler](/cpp/cpp/smart-pointers-modern-cpp) ve [ C++ standart kitaplık](/cpp/standard-library/cpp-standard-library-reference).

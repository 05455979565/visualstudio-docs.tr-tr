# <a name="c6284"></a>C6284
Uyarı C6284: '% d ' parametresi olarak bir dize gerektiği zaman, \<function > çağrısında nesne geçirildi.

 Bu uyarı, biçim dizesinin bir dize (örneğin, `printf` veya `scanf` için `%s` belirtimi) belirtir, ancak bunun yerine bir C++ nesne geçirilir.

 Bu hata yanlış çıkış veya kilitlenme üretebilir.

 Bu ileti genellikle `std::string`, @no__t- C++ 2 veya `bstr_t` gibi bazı dize türlerini uygulayan bir nesnenin bir C @no__t -4-Style çağrısına geçirilmesi nedeniyle raporlanır. C++ Sınıfının uygulamasına bağlı olarak, uygun atama işleçleri tanımlanmışsa dize nesneleri genellikle her C dizesi gerektiğinde saydam olarak C++ kullanılabilir; Ancak, @no__t -2-Style işlevlerine yönelik parametreler temelde türsüz olduğundan, bir dizeye dönüştürme gerçekleşmeyebilir.

 Nesneye bağlı olarak, uygun dize türüne `static_cast` işleci eklemek uygun olabilir (örneğin, `char *` veya `TCHAR``*`) ya da bir dize döndüren bir üye işlevi çağırabilir (örneğin, `std::string` örneklerinde `c_str()`).

## <a name="example"></a>Örnek
 Aşağıdaki kod bu uyarıyı oluşturur çünkü bir `CComBSTR` `sprintf` işlevine geçirilir:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 Aşağıdaki kod bu uyarıyı düzeltmek için statik atama kullanır:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```

# <a name="ca1716-identifiers-should-not-match-keywords"></a>CA1716: Tanımlayıcılar anahtar sözcüklerle eşleşmemelidir

|||
|-|-|
|TypeName|IdentifiersShouldNotMatchKeywords|
|CheckId|CA1716|
|Kategori|Microsoft.Naming|
|Son dakika değişikliği|Yeni|

## <a name="cause"></a>Nedeni

Ad alanı, tür veya sanal veya arabirim üyesinin adı, programlama dilinde ayrılmış bir anahtar kelimeyle eşleşir.

Varsayılan olarak, bu kural yalnızca dışarıdan görünen ad alanlarına, türlerine ve üyelerine bakar, ancak görünürlüğü ve [sembol türlerini](#analyzed-symbol-kinds) [yapılandırabilirsiniz.](#api-surface)

## <a name="rule-description"></a>Kural açıklaması

Ad alanları, türleri ve sanal ve arabirim üyeleri için tanımlayıcılar, ortak dil çalışma süresini hedefleyen diller tarafından tanımlanan anahtar kelimelerle eşleşmemelidir. Kullanılan dile ve anahtar kelimeye bağlı olarak, derleyici hataları ve belirsizlikler kitaplığın kullanımını zorlaştırabilir.

Bu kural, aşağıdaki dillerdeki anahtar kelimelere karşı denetler:

- Visual Basic
- C#
- C++/CLI

Visual Basic anahtar kelimeleri için büyük/küçük harf duyarsız karşılaştırması, diğer diller için ise büyük/küçük harf duyarlı karşılaştırma kullanılır.

## <a name="how-to-fix-violations"></a>İhlalleri düzeltme

Anahtar kelimeler listesinde görünmeyen bir ad seçin.

## <a name="when-to-suppress-warnings"></a>Uyarıları ne zaman bastırır

Tanımlayıcının API kullanıcılarının kafasını karıştırmayeceğinden ve kitaplığın .NET'teki tüm dillerde kullanılabilir olduğuna ikna olursanız, bu kuraldan gelen bir uyarıyı bastırabilirsiniz.

## <a name="configurability"></a>Konfigürasyon

### <a name="api-surface"></a>Api yüzeyi

Bu kuralı [FxCop çözümleyicilerinden](install-fxcop-analyzers.md) (eski çözümlemeyle değil) çalıştırıyorsanız, kod tabanınızın erişilebilirliklerine bağlı olarak bu kuralı hangi bölümlerde çalıştıracakşekilde yapılandırabilirsiniz. Örneğin, kuralın yalnızca genel olmayan API yüzeyine karşı çalışması gerektiğini belirtmek için, projenizdeki bir .editorconfig dosyasına aşağıdaki anahtar değer çiftini ekleyin:

```ini
dotnet_code_quality.ca1716.api_surface = private, internal
```

Bu seçeneği yalnızca bu kural, tüm kurallar veya bu kategorideki tüm kurallar (Adlandırma) için yapılandırabilirsiniz. Daha fazla bilgi için Bkz. [Bkz.](configure-fxcop-analyzers.md)

### <a name="analyzed-symbol-kinds"></a>Çözümlenmiş sembol türleri

Bu kuraltarafından çözümlenecek üye türlerini yapılandırabilirsiniz. Virgül ayrılmış liste `Namespace` `Parameter` olarak `NamedType` `Method`bu `Property` `Event`değerlerden birini veya birkaçını kullanın. Varsayılan değerin yukarıdaki üyelerin tümlerini içerdiğini unutmayın.

```ini
dotnet_code_quality.CA1716.analyzed_symbol_kinds = Namespace, NamedType, Method, Property, Event
```

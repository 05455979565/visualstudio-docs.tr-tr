# <a name="ca1305-specify-iformatprovider"></a>CA1305: IFormatProvider belirtme

|||
|-|-|
|TypeName|SpecifyIFormatProvider|
|CheckId|CA1305|
|Kategori|Microsoft.Globalization|
|Son dakika değişikliği|Kırılmaz|

## <a name="cause"></a>Nedeni

Bir yöntem veya oluşturucu, parametre kabul eden <xref:System.IFormatProvider?displayProperty=fullName> aşırı yüklemeleri olan bir veya daha fazla üyeyi <xref:System.IFormatProvider> çağırır ve yöntem veya oluşturucu, parametreyi alan aşırı yüklemeyi çağırmaz.

Bu kural, parametreyi yoksaydığı belgelenen .NET yöntemlerine <xref:System.IFormatProvider> yapılan çağrıları yok sayar. Kural ayrıca aşağıdaki yöntemleri yok sayar:

- <xref:System.Activator.CreateInstance%2A?displayProperty=nameWithType>
- <xref:System.Resources.ResourceManager.GetObject%2A?displayProperty=nameWithType>
- <xref:System.Resources.ResourceManager.GetString%2A?displayProperty=nameWithType>
- <xref:System.Boolean.ToString%2A?displayProperty=nameWithType>
- <xref:System.Char.ToString%2A?displayProperty=nameWithType>
- <xref:System.Guid.ToString%2A?displayProperty=nameWithType>

## <a name="rule-description"></a>Kural açıklaması

Bir <xref:System.Globalization.CultureInfo?displayProperty=nameWithType> veya <xref:System.IFormatProvider> nesne sağlanmadığında, aşırı yüklenen üye tarafından sağlanan varsayılan değer tüm yerel bölgelerde istediğiniz etkiye sahip olmayabilir. Ayrıca, .NET üyeleri kodunuz için doğru olmayabilir varsayımlara göre varsayılan kültür ve biçimlendirme seçin. Kodun senaryolarınız için beklendiği gibi çalıştığından emin olmak için, kültüre özel bilgileri aşağıdaki yönergelere göre sağlamanız gerekir:

- Değer kullanıcıya görüntülenecekse, geçerli kültürü kullanın. Bkz. <xref:System.Globalization.CultureInfo.CurrentCulture%2A?displayProperty=nameWithType>.

- Değer depolanacak ve yazılım tarafından erişilecekse (bir dosya veya veritabanına kalıcı olarak saklı), değişmez kültürü kullanın. Bkz. <xref:System.Globalization.CultureInfo.InvariantCulture%2A?displayProperty=nameWithType>.

- Değerin hedefini bilmiyorsanız, veri tüketicisinin veya sağlayıcısının kültürü belirtmesi gerekir.

Aşırı yüklenen üyenin varsayılan davranışı gereksinimlerinize uygun olsa bile, kodunuzun kendi kendine belgelenmesi ve daha kolay korunması için kültüre özgü aşırı yüklemeyi açıkça aramak daha iyidir.

## <a name="how-to-fix-violations"></a>İhlalleri düzeltme

Bu kuralın ihlalini düzeltmek <xref:System.IFormatProvider> için, bir bağımsız değişken gerektiren aşırı yüklemeyi kullanın. Veya [C# enterpolasyonlu](/dotnet/csharp/tutorials/string-interpolation) bir dize <xref:System.FormattableString.Invariant%2A?displayProperty=nameWithType> kullanın ve yönteme geçirin.

## <a name="when-to-suppress-warnings"></a>Uyarıları ne zaman bastırır

Varsayılan biçimin doğru seçim olduğundan ve kod bakımının önemli bir geliştirme önceliği olmadığı durumlarda bu kuraldan gelen bir uyarıyı bastırmak güvenlidir.

## <a name="example"></a>Örnek

Aşağıdaki kodda, `example1` dize CA1305 kuralını ihlal eder. Dize, `example2` CA1305 kuralını , <xref:System.Globalization.CultureInfo.CurrentCulture%2A?displayProperty=nameWithType>uygular <xref:System.IFormatProvider>, . <xref:System.String.Format(System.IFormatProvider,System.String,System.Object)?displayProperty=nameWithType> Dize, `example3` interpolated dize 'ye geçirerek CA1305 kuralını <xref:System.FormattableString.Invariant%2A?displayProperty=fullName]>karşılar.

```csharp
string name = "Georgette";

// Violates CA1305
string example1 = String.Format("Hello {0}", name);

// Satisfies CA1305
string example2 = String.Format(CultureInfo.CurrentCulture, "Hello {0}", name);

// Satisfies CA1305
string example3 = FormattableString.Invariant($"Hello {name}");
```

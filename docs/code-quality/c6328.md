# <a name="c6328"></a>C6328

Uyarı C6328: Boyut uyuşmazlığı: \<işlevine yapılan çağrıda >-2Type @no__t gerekliyse, \<type > parametre olarak geçildi \<sayı > >

@No__t-0 türünde bir bağımsız değişkeni C çalışma zamanına geçirme, örneğin `is<xxx>()` adlı karakter tabanlı yordamlar, örneğin `isspace()`, öngörülemeyen sonuçlara neden olabilir. Örneğin, `0x7F` ' den büyük bir değere sahip `char` türünde bir SBCS veya MBCS tek baytlı karakteri negatif bir değerdir. @No__t-0 geçirilirse, derleyici değeri işaretli bir `int` veya imzalı `long` ' e dönüştürebilir. Bu değer, derleyici tarafından beklenmedik sonuçlarla birlikte oturum açma için genişletilebilir. Örneğin `isspace`, `int`; türünde bir bağımsız değişkeni kabul eder. Ancak, giriş bağımsız değişkeni için geçerli değer aralığı:

`0 <= c <= 255`, artı-1 @no__t özel değeri.

## <a name="example"></a>Örnek

Varsayılan olarak, `char`, Visual C++'teki imzalı bir türdür, bu nedenle char türünde bir değişkenin değer aralığı `-128 <= c <= 127` ' dir. Bu nedenle, aşağıdaki işlemleri yaptıysanız, `c` değeri-37 olan işaretli bir `int` ' e, [ısspace](/cpp/standard-library/locale-functions#isspace)için geçerli aralığın dışında bir değer olarak genişletilir:

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( c );
    // code ...
}
```

Bu sorunu düzeltmek için, aşağıdaki kodda gösterildiği gibi `static_cast` kullanabilirsiniz:

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( static_cast<unsigned char> (c) );
    // code ...
}
```

Uyarı C6328 bu hatayı yakalamak için özel olarak mevcuttur. 7 bit ASCII aralığındaki karakterler için, atama gereksizdir. Bu aralığın dışındaki karakterler öngörülemeyen sonuçlara neden olabilir; Örneğin, program hatası ve sonlandırma.

# <a name="c26135"></a>C26135
Uyarı C26135: Eksik ek açıklama \<annotation >, \<func > işlevinde.

 Çözümleyici, bir işlevin bir işlev tarafından bir kilit alma veya kilit verme yan etkisi olan bir kilit sarmalayıcı işlevi olduğu zaman C26135 verilir. Kodun bir sarmalayıcı işlevi olması amaçlanmıyorsa, kilit sızıyor (kilit alınırsa) veya yanlış yayımlanıyorsa (kilit yayımlanıyorsa).

## <a name="example"></a>Örnek
 Aşağıdaki örnek, uygun bir yan etkisi ek açıklaması eksik olduğu için Warning C26135 oluşturur.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

void MyEnter(DATA* p)
{
    // Warning C26135:
    // Missing side effect annotation _Acquires_lock_(&p->cs)
    EnterCriticalSection(&p->cs);
}

void MyLeave(DATA* p)
{
    // warning C26135:
    // Missing side effect annotation _Releases_lock_(&p->cs)
    LeaveCriticalSection(&p->cs);
}
```

## <a name="example"></a>Örnek
 Uyarı C26135, bir koşullu kilitleme yan etkisi algılandığında da yayımlanır. Koşullu bir etkiye açıklama eklemek için `_When_(ConditionExpr, LockAnnotation)` ek açıklamasını kullanın; burada `LockAnnotation` `_Acquires_lock_` veya `_Releases_lock_` ' ü ve koşul ifadesi `ConditionExpr` ' ü Boolean koşullu bir ifadedir. Aynı işlevdeki diğer ek açıklamaların yan etkileri yalnızca `ConditionExpr` doğru olarak değerlendirildiğinde oluşur. @No__t-0, koşulu çağırana geri geçirmek için kullanıldığından, çağıran bağlamda tanınan değişkenleri içermelidir. Bunlar işlev parametrelerini, genel veya sınıf üyesi değişkenleri ya da dönüş değerini içerir. Dönüş değerini görmek için, aşağıdaki örnekte gösterildiği gibi, `return` ek açıklamasında özel bir anahtar sözcük kullanın.

```

 typedef struct _DATA
 {
     CRITICAL_SECTION cs;
     int state;
} DATA;

 _When_(return != 0, _Acquires_lock_(p->cs))
 int TryEnter(DATA* p)
 {
     if (p->state != 0)
     {
         EnterCriticalSection(&p->cs);
         return p->state;
     }

     return 0;
 }
```

 Okuyucu/Yazıcı kilitleri olarak da bilinen paylaşılan/özel kilitler için aşağıdaki ek açıklamaları kullanarak, yan etkileri de genişletebilirsiniz:

- `_Acquires_shared_lock_(LockExpr)`

- `_Releases_shared_lock_(LockExpr)`

- `_Acquires_exclusive_lock_(LockExpr)`

- `_Releases_exclusive_lock_(LockExpr)`

# <a name="c6401"></a>C6401
Uyarı C6401: sabit dize \< dize adı > için büyük/küçük harfe duyarsız bir karşılaştırma gerçekleştirmek üzere varsayılan bir yerel ayarda \<işlev adı > kullanılıyor. Ingilizce olmayan yerel ayarlarda beklenmeyen sonuçlar verir

 Bu uyarı, varsayılan yerel ayarı belirtirken sabit bir dizeye yönelik büyük/küçük harfe duyarsız bir karşılaştırmanın gerçekleştirildiğini belirtir; Genellikle, yerel ayara bağımsız bir karşılaştırma amaçlandı.

 Bu hatanın tipik sonucu, Ingilizce olmayan konuşma yerel ayarları 'nda yanlış davranıştır. Örneğin, Türkçe 'de ". gif" eşleşmeyecektir ". GIF "; Vietnam dilinde "LogIn", "LOGIN" ile eşleşmeyecektir.

 @No__t-0 işlevi bir yerel ayarı bağımsız değişken olarak alır; Bununla birlikte, varsayılan bir yerel ayarı geçirmek, örneğin `LOCALE_USER_DEFAULT` sabiti, kullanıcının varsayılan ayarlarına bağlı olarak farklı yerel ayarlarda farklı davranışlara neden olur. Genellikle, sabit bir dizeye karşılık gelen büyük/küçük harfe duyarsız karşılaştırmalar, yerel ayara bağımsız bir karşılaştırmada gerçekleştirilmelidir.

 Windows XP 'de `CompareString` kullanarak yerel ayara bağımsız bir karşılaştırma gerçekleştirmek için, ilk parametre sabit `LOCALE_INVARIANT` olmalıdır; Örneğin, `pString` ' nin, büyük/küçük harf farklılıklarını yoksayarak FILE1. gif ile eşleşip eşleşmediğini gösteren bir yerel ayara bağımsız test gerçekleştirmek için, şöyle bir çağrı kullanın:

```
CompareString(LOCALE_INVARIANT,
              NORM_IGNORECASE,
              pString,
              -1,
              TEXT("file1.gif"),
              -1) == CSTR_EQUAL
```

## <a name="example"></a>Örnek
 Aşağıdaki kod bu uyarıyı oluşturur:

```
include <windows.h>

int fd(char *ext)
{
  return (CompareString(LOCALE_USER_DEFAULT,
                        NORM_IGNORECASE,
                        ext,
                        -1,
                        TEXT("gif"),
                        -1) == 2);
}
```

 Bu uyarıyı düzeltmek için aşağıdaki kodu kullanın:

```

{

```

# <a name="c26110"></a>C26110
Uyarı C26110: işlev \<func > çağrılmadan önce, çağıran \<lock > tutmada başarısız oldu.

 Bir kilit gerektiğinde, işlevin kendisinin mi yoksa arayanın mi elde etmeyeceğini açıklığa kavuşturduğunuzdan emin olun. Uyarı C26110, `_Requires_lock_held_` ek açıklaması veya kilit ile ilgili diğer ek açıklamaları ihlal edildiğinde verilir. Daha fazla bilgi için bkz. [kilitleme davranışına açıklama ekleme](annotating-locking-behavior.md)

## <a name="example"></a>Örnek
 Aşağıdaki örnekte, `_Requires_lock_held_` işlevindeki `LockRequired` ek açıklaması, `LockRequired` ' i çağırmadan önce `LockRequired` çağıranın kilidi edinmesi gerektiğini belirten uyarı C26110 oluşturulur. Bu ek açıklama olmadan, `LockRequired` ' ın kilit tarafından korunan paylaşılan verilere erişmesi için kilidi almaları gerekir.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
    int d;
} DATA;

_Requires_lock_held_(p->cs)

void LockRequired(DATA* p)
{
    p->d = 0;
}

void LockNotHeld(DATA* p)
{
    LockRequired(p); // Warning C26110
}
```

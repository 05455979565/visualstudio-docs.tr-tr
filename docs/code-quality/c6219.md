# <a name="c6219"></a>C6219

Uyarı C6219: anlamsal olarak farklı tamsayı türleri arasında örtük atama: HRESULT, 1 veya TRUE ile karşılaştırılıyor. Bunun yerine [başarılı](/windows/desktop/api/winerror/nf-winerror-succeeded) veya [başarısız](/windows/desktop/api/winerror/nf-winerror-failed) makro kullanmayı düşünün

Bu uyarı, bir `HRESULT` ' ın açık, @no__t olmayan bir değer (1) ile karşılaştırıldığını gösterir. @No__t-0 ' ın (`S_OK`) tipik başarı değeri 0 olduğundan, bu karşılaştırma yanlış sonuçlara neden olabilir. Bu değeri bir Boole türüyle karşılaştırırsanız, örtülü olarak false değerine dönüştürülür.

## <a name="example"></a>Örnek

Aşağıdaki kod bu uyarıyı oluşturur çünkü `CoGetMalloc` `HRESULT` döndürdüğünden `TRUE` ile karşılaştırılır:

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (hr == TRUE)
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

Çoğu zaman, bu uyarı `HRESULT` ' ı bir Boole ile karşılaştıran koddan oluşur. Bir `HRESULT` değerini test etmek için `SUCCEEDED` veya `FAILED` makrolarını kullanmak daha iyidir. Bu uyarıyı düzeltmek için aşağıdaki kodu kullanın:

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

Bu uyarı için `SCODE` türü `HRESULT` olarak değerlendirilir.

@No__t-0 ve `free` (ve ilgili dinamik bellek API 'Leri) kullanımı, bellek sızıntıları ve özel durumların bir nedeni olarak çok sayıda gizli dizi içerir. Bu tür sızıntıları ve özel durum sorunlarını önlemek için C++ standart kitaplık tarafından sunulan işaretçi ve kapsayıcı sınıflarını kullanın. Bunlar şunlardır [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)ve [Vector](/cpp/standard-library/vector). Daha fazla bilgi için bkz. [akıllı işaretçiler](/cpp/cpp/smart-pointers-modern-cpp) ve [ C++ standart kitaplık](/cpp/standard-library/cpp-standard-library-reference).

# <a name="c6273"></a>C6273
Uyarı 6273-tamsayı olmayan bir parametre olarak geçildi \<number > \<işlev > çağrısında tamsayı gerekliyse: bir işaretçi değeri geçiriliyorsa,% p kullanılmalıdır

 Bu uyarı, biçim dizesinin bir tamsayıyı (örneğin, `printf` için `%d`, uzunluk veya öncelik belirtimini), ancak `float`, dize veya `struct` gibi tamsayı olmayan bir parametre olarak geçtiğini gösterir. Bu hata, büyük olasılıkla hatalı çıktıya neden olabilir.

## <a name="example"></a>Örnek
 Aşağıdaki kod bu uyarıyı oluşturur çünkü `float`-`sprintf`işlevi yerine bir tamsayı gereklidir:

```
#include <stdio.h>
#include <string.h>

void f_defective()
{
  char buff[50];
  float f=1.5;

  sprintf(buff, "%d",f);
}
```

 Aşağıdaki kod bu uyarıyı düzeltmek için bir tamsayı atama kullanır:

```
#include <stdio.h>
#include <string.h>

void f_corrected()
{
  char buff[50];
  float f=1.5;

  sprintf(buff,"%d",(int)f);
}
```

 Aşağıdaki kod, bu uyarıyı düzeltmek için `sprintf_s` olan güvenli dize işleme işlevini kullanır:

```
#include <stdio.h>
#include <string.h>

void f_safe()
{
  char buff[50];
  float f=1.5;

  sprintf_s(buff,50,"%d",(int)f);
}
```

 % P bu platformlarda desteklenmediğinden, bu uyarı Windows 9x ve Windows NT sürüm 4 ' te geçerli değildir.

# <a name="c26105"></a>C26105
Uyarı C26105: Kilit sırası ihlali. Kilit @no__t alma-0 \<seviye > ile 0lock > Order Inversion 'un oluşmasına neden olur.

 Eşzamanlılık SAL, *kilit düzeylerini*destekler. Çift tırnak olmadan bir dize sabit değeri tarafından belirtilen bir kilit düzeyi bildirmek için `_Create_lock_level_` kullanın. @No__t-0 ' ını kullanarak iki kilit düzeyi arasında bir alım siparişi uygulayabilirsiniz. Bu, düzey @no__t düzeyi olan kilitlerin önüne `A` olan kilitlerin elde edilmesi gerektiğini belirtir. Kilit sırası hiyerarşisi oluşturmak için (kilit düzeyleri arasında kısmi bir sıra), birden çok `_Set_lock_level_order_` ek açıklaması kullanın. Kilidi bir kilit düzeyiyle ilişkilendirmek için, kilidi bildirdiğinizde `_Set_lock_level_` ek açıklamasını kullanın. Bir kilit sıralaması ihlali algılandığında uyarı C26105 verilir.

## <a name="example"></a>Örnek
 Aşağıdaki örnek, `OrderInversion` işlevinde bir kilit sırası Inversion olduğundan, uyarı C26105 oluşturur.

```

_Create_lock_level_(MutexLockLevel);
_Create_lock_level_(TunnelLockLevel);
_Create_lock_level_(ChannelLockLevel);
_Lock_level_order_(MutexLockLevel, TunnelLockLevel);
_Lock_level_order_(TunnelLockLevel, ChannelLockLevel);
_Has_lock_level_(MutexLockLevel) HANDLE gMutex;

struct Tunnel
{
    _Has_lock_level_(TunnelLockLevel) CRITICAL_SECTION cs;
};

struct Channel
{
    _Has_lock_level_(ChannelLockLevel) CRITICAL_SECTION cs;
};

void OrderInversion(Channel* pChannel, Tunnel* pTunnel)
{
    EnterCriticalSection(&pChannel->cs);
    // Warning C26105
    WaitForSingleObject(gMutex, INFINITE);
    EnterCriticalSection(&pTunnel->cs);
    LeaveCriticalSection(&pTunnel->cs);
    LeaveCriticalSection(&pChannel->cs);
}
```

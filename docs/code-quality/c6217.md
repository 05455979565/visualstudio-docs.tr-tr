# <a name="c6217"></a>C6217

Uyarı C6217: anlamsal olarak farklı tamsayı türleri arasında örtük atama: HRESULT ' Not ' ile test ediliyor. Bunun yerine [başarılı](/windows/desktop/api/winerror/nf-winerror-succeeded) veya [başarısız](/windows/desktop/api/winerror/nf-winerror-failed) makro kullanmayı düşünün.

Bu uyarı, bir `HRESULT` ' ın Not (`!`) işleci ile test edildiğini gösterir. @No__t-1 ' deki başarılı (`S_OK`) değeri 0 ile belirtilir. Ancak, 0 Boole türü hatası gösterir. Hangi kod bloğunun çalıştırılacağını anlamak için Not işleci (`!`) ile @no__t test etmek, yanlış kod yolunun takip edilmesine neden olabilir. Bu, istenmeyen sonuçlara yol açacaktır.

## <a name="example"></a>Örnek

Aşağıdaki kod bu uyarıyı oluşturur çünkü Not işleci `HRESULT` değerinin başarısını veya başarısızlığını tespit etmek için kullanılır. Bu durumda, `( !hr )` hata kodunu çalıştırdığı için yanlış kod yolu yürütülür:

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (!hr)
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

Bu uyarıyı düzeltmek için aşağıdaki kod hata aramak için `FAILED` makrosunu kullanır:

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (FAILED(hr))
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

Bu uyarı için `SCODE` türü `HRESULT` ' e eşdeğerdir.

HRESULT (`S_OK`) için tipik başarı değeri, Boolean olarak test edildiğinde `false` ' dir.

@No__t-0 ' ın başarılı olup olmadığını doğrulamak için, bunun yerine `SUCCEEDED` makrosunu kullanın.

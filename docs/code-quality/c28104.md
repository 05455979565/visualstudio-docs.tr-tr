# <a name="c28104"></a>C28104
Uyarı C28104: İşlev çıkmadan önce alınması gereken kaynak alınmadı

 Uygulamadan çıkmadan önce bir kaynağın elde edilmesine yönelik bir işlev, kaynağı almadan çıkış yaptı. Bu uyarı, işlevin `__drv_acquiresResource` ile açıklandığını ancak kaynağı gerçekten almış olduğunu döndürmediğini belirtir. Bu işlev bir sarmalayıcı işlevidir, işlev içindeki bir yol Sarmalanan işleve ulaşmadı. Sarmalanan işleve erişim hatası, işlevin bir hata döndürdüğünden ve kaynağı gerçekten elde edemediği için, koşullu ek açıklama (`__drv_when`) kullanmanız gerekebilir.

 Bu işlev kaynak alımı gerçekten uygularsa, PFD 'in kaynağın elde edilen olduğunu algılaması mümkün olmayabilir. Bu durumda, hatayı bastırmak için `#pragma` uyarısı kullanın. Büyük olasılıkla `#pragma` ' yı, işlev gövdesini Başlatan `{` ' den önceki satıra yerleştirebilirsiniz. Çağırma işlevlerinde ek açıklama hala gereklidir, ancak kod çözümleme aracı kaynağın elde edilen olduğunu algılayamayacak.

## <a name="example"></a>Örnek

```
__drv_acquireResourceGlobal(HWLock, lockid)
void GetHardwareLock(lockid)
#pragma warning (suppress: 28104)
{
   // code to implement a hardware lock (which the Code Analysis tool can't recognize)
}
```

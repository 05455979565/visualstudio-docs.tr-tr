# <a name="c6290"></a>C6290
Uyarı C6290: mantıksal sonuç üzerinde bit düzeyinde işlem:! & kıyasla daha yüksek önceliğe sahiptir. & & Veya (!) kullanın ( x & y)) bunun yerine

Bu uyarı, bir operatör veya işleç önceliği kullanmanın olası karışıklık olduğunu gösterir.

@No__t-0 işleci bir Boole sonucu verir ve `&` ' den daha yüksek önceliğe sahiptir. Bit düzeyinde and (&) işleci iki aritmetik bağımsız değişken alır. Bu nedenle, aşağıdaki hatalardan biri algılandı:

- İfade yanlış parantez içine yerleştirildi:

   @No__t-0 ' nin sonucu Boolean (sıfır veya bir) olduğundan, bu iki değişkeni test etme girişimi yalnızca en düşük bitin sağ tarafta mevcut olduğunu test eder: `((!8) & 1) == 0`.

- @No__t-0 işleci yanlış ve bunun yerine bir `~` olmalıdır:

   @No__t-0 işlecinin Boolean sonucu vardır, ancak `~` işlecinin aritmetik bir sonucu vardır. Bu işleçler, bir Boole değeri (sıfır veya bir) üzerinde çalışırken bile hiçbir zaman yoktur: `((!0x01) & 0x10) == 0x0`, `((~0x01) & 0x10) == 0x10`.

- @No__t-0 ikili işleci yanlış ve bunun yerine `&&` olması gerekir:

   @No__t-0 bazen `&&` ile karşılıklı olarak değiştirilebiliyorsa, ifadenin sağ tarafının değerlendirmesini zorlarken bu eşit değildir. Bu tür bir ifade içindeki bazı yan etkileri Terminal olabilir.

Kodu incelemeden bu sorunun önem derecesini kararlaştırmak zordur. Hedeflenen testin gerçekleşdiğinden emin olmak için kodun incelenebilmelidir.

## <a name="example"></a>Örnek

Aşağıdaki kod bu uyarıyı oluşturur:

```cpp
void f(int x, int y)
{
  if (!x & y)
  {
    // code ..
  }
}
```

Bu uyarıyı düzeltmek için aşağıdaki örnek kodu kullanın:

```cpp
void f(int x, int y)
{
  /* When testing that x has no bits in common with y. */
  if (!(x & y))
  {
    // code
  }

  /* When testing for the complement of x in y. */
  if ((~x) & y)
  {
    // code ...
  }
}
#include <windows.h>
void fC(int x, BOOL y )
 {
  /* When y is a Boolean or Boolean result. */
  if ((!x) && y)
  {
    // code ...
  }
}
```

Uyarı C6317, `!` işleci `&` işlecinin sağ tarafında ise raporlanır.

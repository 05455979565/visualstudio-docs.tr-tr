# <a name="execution-control-and-state-evaluation"></a>Yürütme denetimi ve durum değerlendirme
Bir uygulamada hata ayıklama işlevleri Adımlama, kesme noktalarında durdurma ve yürütmeye devam etme gibi yürütme denetim özelliklerini uygulama gerektirir. Tabanları hata ayıklama visual Studio hata ayıklayıcısı bileşenler arasında kendi yürütme denetim olaylarına gönderdi.  
  
## <a name="in-this-section"></a>Bu Bölümde  
 [Program Denetimi](../../extensibility/debugger/program-control.md)  
 Program düzeyinde meydana geldiği aşağıdaki yordamları listeler: next deyimi ayarlama, yürütme, Adımlama, devam etmeden, askıya alma ve sürdürme.  
  
 [Kesme Noktasıyla İlgili Metotlar](../../extensibility/debugger/breakpoint-related-methods.md)  
 Bağımlı tanımlar ve Visual Studio destekler kesme noktaları tür bekleniyor.  
  
 [Çağrı Yığını Değerlendirme](../../extensibility/debugger/call-stack-evaluation.md)  
 Çağrı yığınının yığın çerçeveleri görüntüleme kesme modu sırasında izin yöntemleri uyarlamasını açıklanır.  
  
 [İfade değerlendirme](../../extensibility/debugger/expression-evaluation-visual-studio-debugging-sdk.md)  
 Hata ayıklama altyapısı (DE) nasıl ifade değerlendirme (EE) ve oturum hata ayıklama Yöneticisi ayrıştırma dahil olan ve bir ifadenin değerlendirmesine IDE windows birine girilen açıklanmaktadır.  
  
 [Denetim Olayları](../../extensibility/debugger/control-events.md)  
 Denetimli program yürütülmesi sırasında olayları göndermek için kullanılan arabirimini tartışmaktadır.
